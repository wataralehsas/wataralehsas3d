## خطة التطوير الشاملة

### 1. توحيد جماليّة "بستان الورد" عبر كل الأقسام
- توحيد رؤوس الصفحات (Hero) لكل قسم بنفس نمط Rose Garden مع لون مميّز لكل قسم:
  - **قصة الشعر** (`/haircut`): وردي دافئ
  - **التجربة الافتراضية** (`/tryon`): بنفسجي وردي
  - **المعرض** (`/marketplace`): ذهبي وردي
  - **المحاكي** (`/simulator`): وردي فاتح
  - **أدوات التسويق** (`/marketing-tool`): وردي ذهبي
- إضافة مكوّن مشترك `<SectionHero />` يضمن نفس البطاقة الزجاجية، البتلات المتساقطة، ومؤشّر المحاولات المتبقية.

### 2. تحسينات قسم قصة الشعر (والأقسام التوليدية)
- **التقاط مباشر من الكاميرا**: زر "التقط الآن" يفتح كاميرا الهاتف عبر `<input type="file" accept="image/*" capture="environment">` + معاينة فورية.
- **إضافة تصميم/قصّة جديدة** من الواجهة: حقل اسم + برومبت إنجليزي + معاينة، يُحفظ في إعدادات المنصّة (`settings.customHaircuts[]`) ويظهر مباشرة في قائمة الاختيار.
- **حفظ الصورة المولّدة على الهاتف**:
  - زر "حفظ في المعرض" يستخدم Web Share API (`navigator.share` مع `files`) على الجوال؛ يسقط تلقائياً إلى تنزيل `download` على المتصفّحات الأخرى.
  - زر إضافي "نسخ الصورة" + "مشاركة واتساب" بعد التوليد.
- نفس النمط يُطبَّق على `/tryon` و`/marketing-tool` لاحقاً.

### 3. نظام الاشتراكات المدفوعة عبر شام كاش
- **إعدادات الأدمن الجديدة** في `/admin` → تبويب جديد "الاشتراكات والدفع":
  - تفعيل/إيقاف الاشتراكات المدفوعة.
  - باقات قابلة للتعديل: `{ id, attempts, priceUSD, priceSYP, label }` (افتراضي: 100 محاولة بـ 5$، 250 بـ 10$، 600 بـ 20$).
  - رفع صورة QR الخاصة بشام كاش (تُحفَظ كـ DataURL في `settings`).
  - رقم شام كاش + اسم المستلِم + تعليمات إضافية.
- **واجهة الزائر** (تظهر داخل `QuotaModal` عند نفاد المحاولات):
  - قائمة بالباقات مع الأسعار.
  - عند الاختيار: عرض QR + رقم + زرّ "نسخ الرقم" + حقل إدخال رقم عملية شام كاش + اسم/هاتف المستخدم + زر "أرسلت الدفع".
  - عند الإرسال: إنشاء طلب اشتراك بـ `pending_payments` (Cloud) + إشعار محلي للأدمن + توليد رمز جهاز `deviceId` يُحفَظ في `localStorage` لربط الموافقة لاحقاً.
- **لوحة الأدمن**: تبويب "طلبات الدفع المعلّقة" يعرض القائمة، زر "موافقة" يفتح محاولات الجهاز، زر "رفض" مع سبب.
- **آلية فتح المحاولات**: عند الموافقة، نضيف رصيداً مدفوعاً `paidAttempts` لمعرّف الجهاز عبر Realtime من Cloud؛ `quota.ts` يحسبه قبل الحد المجاني.

### 4. التفاصيل التقنية
- **Cloud Schema** (هجرة جديدة):
  - `payment_packages` (id, attempts, price_usd, price_syp, label, active)
  - `payment_requests` (id, device_id, package_id, user_name, phone, tx_ref, status, created_at, approved_at)
  - `device_credits` (device_id, paid_attempts_remaining, updated_at)
  - RLS: قراءة عامة للباقات، إدراج عام لطلبات الدفع، تعديل/قراءة للأدمن فقط (دور `admin` عبر `has_role`).
- **ملفات جديدة**:
  - `src/components/SectionHero.tsx` — رأس موحّد.
  - `src/components/CameraCapture.tsx` — التقاط الكاميرا.
  - `src/lib/save-image.ts` — حفظ/مشاركة الصورة على الجوال.
  - `src/lib/payments.ts` — منطق الباقات والطلبات.
  - `src/components/PaymentModal.tsx` — واجهة الدفع شام كاش + QR.
  - `src/components/admin/PaymentsTab.tsx` — إدارة الباقات والطلبات.
  - `src/components/admin/HaircutDesignsTab.tsx` — إضافة قصّات شعر مخصّصة.
- **ملفات تُعدَّل**: `quota.ts` (دمج `paidAttempts`)، `QuotaModal.tsx` (زرّ شراء)، `haircut.tsx` (الجماليات + الكاميرا + الحفظ + قصّات الأدمن)، `admin.tsx` (تبويبات جديدة)، `settings.ts` (إعدادات الباقات + QR).

### 5. مرحلة التنفيذ
بسبب حجم المهمّة، سأنفّذها على مرحلتين:
1. **المرحلة A**: الجماليّات الموحّدة + الكاميرا + الحفظ على الهاتف + إضافة قصّات الأدمن (واجهة فقط، بدون Cloud).
2. **المرحلة B**: نظام شام كاش الكامل (Cloud schema + UI الدفع + لوحة موافقات الأدمن + ربط المحاولات).

هل أبدأ بالمرحلة A مباشرةً ثم أتابع B تلقائياً، أم تفضّل B أولاً؟
